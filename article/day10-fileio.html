一、File类：

​ 1）java.io.File(类)：

  • 此类可以封装一个“路径”，这个路径可以代表一个“文件夹/文件”，或者一个“文件”。File类也包含了很多的方法，可以通过这些方法，获取这个“目录/文件“的一些属性信息：文件大小、最后修改时间、是否可读、是否可写……

  • ​ 还可以创建文件/目录，删除文件/目录。

  • ​ 他也是IO流的基础类。（表示一个路径，这个路径存在或不存在，删除File或者创建File都不会对真实路径产生影响，参数可以是文件，也可是路径）

​ 2)构造方法

    1).File(String pathname) ：通过将给定的路径名的字符串转换成一个抽象路径名创建一个新的 File实例。  
                绝对路径：带盘符的全路径；
                相对路径：不带盘符的路径，从项目根目录开始找。
    2).File(String parent, String child) 创建从父路径名的字符串和一个孩子的一个新的 File实例文件。 
    3).File(File parent, String child) 创建从一个家长的抽象路径名和一个孩子的路径字符串的新 File实例。

​ 3）成员方法：

​ A)File类获取相关的方法

1).public String getAbsolutePath() ：返回此File的绝对路径名字符串。

2).public String getPath() ：将此File转换为路径名字符串。 

3).public String getName()  ：返回由此File表示的文件或目录的名称。  

4).public long length()  ：返回由此File表示的文件的大小。 不能获取目录的大小。

​ B).File类判断相关的方法

5).public boolean exists() ：此File表示的文件或目录是否实际存在。

6).public boolean isDirectory() ：此File表示的是否为目录。

7).public boolean isFile() ：此File表示的是否为文件。

​ C)File类判断相关的方法

8).public boolean createNewFile() ：当且仅当具有该名称的文件尚不存在时，创建一个新的空文件。 
    
9).public boolean delete() ：删除由此File表示的文件或目录。  
        
10).public boolean mkdir() ：创建由此File表示的目录。
        
11).public boolean mkdirs() ：创建由此File表示的目录，包括任何必需但不存在的父目录。

​ D)File类目录遍历的相关方法

12).public String[] list() ：返回一个String数组，表示该File目录中的所有子文件或目录。

13).public File[] listFiles() ：返回一个File数组，表示该File目录中的所有的子文件或目录。

​ 1）递归：指方法的”递归调用“——就是方法自己调用自己

  1. 递归的层次不能太深，否则：栈溢出；
  2. 递归一定要有出口，否则就是死递归；

​ 3）递归求阶层：从未知推导到已知（递归调用），从已知往回计算（递归返回）

​ 例如：求5的阶层，可以表示为：5！=5 * 4！

​ 4! = 4 * 3! ​ 3! = 3 * 2! ​ 2! = 2 * 1!

三、IO流：

​ 1）什么是IO流：

​ I：指：Input(输入)：从磁盘到内存，也称作：读取

​ O：指Output（输出）：从内存到磁盘，也称作：写入

​> 流：类似水流，线性的，有方向的。

​ 2）作用：用于读写”文件“的内容

​ 3）Java中的IO流分类：

​ 1）字节流：最基本，最底层的流，可以读写任何文件（纯文本文件、图片、视频、音频。。。）

​ 1）输出流：OutputStream(抽象类)——是所有字节输出流的父类

  • ​ FileOutputStream（子类：文件输出流）

​ 2）输入流：InputStream——是所有字节输入流的父类

  • FileInputStream（子类：文件输入流）

​ 2）字符流：基于字节流，只能读写纯文本文件。

​ 1）输出流：Writer（抽象类）——是所有字符输出流的父类

​ 2）输入流：Reader（抽象类）——是所有字符输入流的父类
